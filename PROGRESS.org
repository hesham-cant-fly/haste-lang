#+TITLE: Haste Language — Overview
#+DATE: 2026-05-14

* Table Of Content :TOC:

* Introduction

This document is an overview of the Haste programming language. You are assumed to have basic knowledge of programming concepts such as variables, statements, and types.

Haste is a compiled, statically-typed language with struct types, compile-time evaluation, and LLVM code generation. It is designed to be simple and expressive.

* Hello, Haste!

To begin, here is a simple Haste program:

#+BEGIN_SRC haste
const greeting: string = "hello";
const year: int = 2026;
#+END_SRC

Save this to a =.haste= file and compile it with the Haste compiler:

#+BEGIN_SRC text
haste file.haste               # compile silently (no output on success)
haste --llvm file.haste         # emit LLVM IR to file.ll
haste --llvm --dump file.haste  # emit LLVM IR to stderr
#+END_SRC

The =--llvm= flag produces LLVM IR output. Without it, the compiler checks the program and exits silently.

* Comments

Comments can be anywhere outside of a string literal. Single-line comments begin with =//= and multi-line comments are enclosed in =/*= and =*/=.

#+BEGIN_SRC haste
// A single-line comment

/* A multi-line
   comment */
#+END_SRC

Multi-line comments do not nest.

* Declarations

** =const= — Immutable binding

A constant declaration introduces an immutable binding. The value may be given after = ==, or omitted if the type is specified — in that case the variable is initialized to the default for that type.

#+BEGIN_SRC haste
const name = value;
const name: type = value;
const name: type;
#+END_SRC

#+BEGIN_SRC haste
const a: int = 42;
const b: float = 3.14;
const c: string = "hello";
const d: int;       // 0
const e: float;     // 0.0
#+END_SRC

When =const= is paired with a =type= value, it creates a type alias:

#+BEGIN_SRC haste
const Number = int;
const a: Number = 42;
#+END_SRC

All =const= declarations are evaluated at compile time when their values are literal or constant-foldable.

** =var= — Mutable binding

#+BEGIN_SRC haste
var name = value;
var name: type = value;
var name: type;
#+END_SRC

=var= uses the same syntax as =const= but produces a mutable binding. At global scope, only =const= is currently supported.

** Untyped inference with =auto=

Using =auto= in type position infers the type from the initializer:

#+BEGIN_SRC haste
const x: auto = 42;    // x is int
const y: auto = 3.14;  // y is float
const z: auto = "hi";  // z is string
#+END_SRC

* Literals

** Numeric literals

Numerical literals are written similarly to most other programming languages. A number containing a dot is a floating-point literal. A leading zero does not produce an octal constant.

#+BEGIN_SRC haste
42        // untyped integer
3.14      // untyped float
0         // zero literal, assignable to any type
#+END_SRC

Untyped numeric literals implicitly convert to typed types on assignment.

** String literals

String literals are enclosed in double quotes. Special characters are escaped with a backslash.

#+BEGIN_SRC haste
"hello world"
""
#+END_SRC

The following escape sequences are supported:

| Sequence | Result |
|----------+--------|
| =\n= | newline |
| =\t= | tab |
| =\r= | carriage return |
| =\0= | null byte |
| =\\= | backslash |
| =\"= | double quote |

* Basic types

** Primitive types

| Type | Description | Default |
|------+-------------+---------|
| =int= | Signed 32-bit integer | =0= |
| =uint= | Unsigned 32-bit integer | =0= |
| =intN= | Signed N-bit integer (e.g., =int8=, =int17=). 1 <= N <= 128 | =0= |
| =uintN= | Unsigned N-bit integer (e.g., =uint8=). 1 <= N <= 128 | =0= |
| =float= | 32-bit float | =0.0= |
| =usize= | Unsigned 64-bit size integer | =0= |
| =string= | String (struct with ptr + len) | `{ ptr: "", len: 0 }` |
| =cstr= | Null-terminated C string pointer | =NULL= |
| =void= | No value | — |
| =type= | The type of types | — |

*** =string= internals

=string= is not a primitive at the implementation level — it is a =struct= type defined at compile-time as:

#+BEGIN_SRC haste
struct { ptr: cstr; len: usize; }
#+END_SRC

- =ptr= points to the string data (a C-string pointer).
- =len= is the byte length of the string (unsigned 64-bit).

=string= is constructed during builtin initialization by the compiler itself. This means =string= participates in all struct machinery: member access, casting, and struct type comparison. Conversions between =string= and =cstr= copy or borrow the internal pointer accordingly.

*** =usize=

=usize= is an unsigned 64-bit integer type used for sizes, lengths, and indices. It is not implicitly convertible from/to =int= — explicit =cast= is required.

#+BEGIN_SRC haste
const a: usize = 42;         // typed literal
const b: int = cast[int]a;   // usize → int (explicit)
const c: usize = cast[usize]b;
#+END_SRC

=usize= is used as the type of =string.len= and appears in struct field contexts where a size is needed.

*** Variable Bit-Width Integers

Haste supports dynamic bit-width integer types using the =intN= and =uintN= syntax, where =N= is any integer between 1 and 128 (inclusive):

#+BEGIN_SRC haste
const byte: uint8 = 255;
const odd_bits: int17 = 42;
const huge: int128 = 0;
#+END_SRC

- Bit-widths must fall strictly within the range `1 <= N <= 128`. Types like =int0= or =int256= will trigger a compile-time bounds error.
- Arithmetic and casting between arbitrary bit-widths behaves identically to standard =int= properties, seamlessly expanding or truncating integers primarily through explicit =cast[TargetType]= operations.

** Untyped types

Certain expressions have an "untyped" type. An untyped type can implicitly convert to a typed type.

| Type | Example | Converts to |
|------+---------+-------------|
| =untyped_int= | =42= | =int=, =float=, =usize= |
| =untyped_float= | =3.14= | =float= |
| =untyped_string= | ="hello"= | =string=, =cstr= |

** Zero values

Variables declared without an explicit initial value are given their zero value: =0= for =int=, =0.0= for =float=, and =""= (empty string) for =string= and =cstr=.

* Type conversion

The =cast= operator converts a value from one type to another:

#+BEGIN_SRC haste
const a: float = 3.14;
const b: int = cast[int]a;           // 3
const c: float = cast[float]42;      // 42.0

const d: string = cast[string]"hello";
const e: cstr = cast[cstr]"world";
const f: string = cast[string]e;     // cstr → string
const g: cstr = cast[cstr]d;         // string → cstr
#+END_SRC

The following conversions are allowed:

| Source | Target |
|--------+--------|
| =untyped_int= | =int=, =float= |
| =untyped_float= | =float=, =int= |
| =int= | =float= |
| =float= | =int= |
| =untyped_string= | =string=, =cstr= |
| =string= | =cstr= |
| =cstr= | =string= |
| =auto_struct= | =struct= (by name-matched fields) |

** Automatic casting
When the target type can be inferred from context (e.g., from a variable's type annotation or a struct field's declared type), the bracket syntax may be omitted:

#+BEGIN_SRC haste
const a: float = 6.7;
const b: int = cast a;       // inferred from `b: int`
const c: string = "world";
const d: cstr = cast c;      // inferred from `d: cstr`
const e: int = cast a + cast b; // inferred from `e: int`
#+END_SRC

The inferred target type propagates through the following contexts:
- Variable declaration type annotation (=cast expr= inside =const x: T = ...= or =var x: T = ...=)
- Struct field default value (=field: T = cast expr= inside a =struct= definition)
- Struct literal field value (=field: cast expr= inside a =Type{...}= literal)

you can also use =cast[auto]= and =cast[]= (with nothing inside)

This also works with auto-struct literals, where the target struct type is inferred:

#+BEGIN_SRC haste
const Job = struct {
    name: string;
};
const Person = struct {
    name: string;
    job: Job;
};

const me: Person = cast .{
    name: "Hesham",
    job: Job{ name: "Coder" },
};
#+END_SRC

The auto-struct =.{...}= fields are matched by name to =Person='s fields, and the result is cast to =Person=.

* Operators

** Unary operators

#+BEGIN_SRC haste
+expression
-expression
#+END_SRC

** Binary operators

#+BEGIN_SRC haste
left + right
left - right
left * right
left / right
#+END_SRC

When both operands are typed but of different types, the compiler rejects the expression — =int + float= is an error. If both operands are untyped, the result takes the wider untyped type: =untyped_int + untyped_float → untyped_float=. When one operand is typed and the other is untyped, the untyped value coerces to the typed type. Division by zero is caught at compile time.

** Assignment compatibility

| Declared type | Accepts |
|---------------+---------|
| =int=, =uint= | =int=, =uint=, =untyped_int= |
| =intN=, =uintN= | Matching integer type, =untyped_int= |
| =usize= | =usize=, =untyped_int= |
| =float= | =float=, =untyped_int=, =untyped_float= |
| =string= | =string=, =untyped_string= |
| =cstr= | =cstr=, =untyped_string= |
| =auto= | anything |

* Struct types

A struct type groups several fields under a single name. Each field is declared as =name: type;= with an optional = = default value.

#+BEGIN_SRC haste
const Vec2 = struct {
    x: float;
    y: float = 0;   // defaults to 0.0 when omitted
};
#+END_SRC

A struct type is itself a =type= value — =typeof(Vec2)= returns =type=. Struct types are nominally typed, meaning each struct declaration creates a unique type. =S1 = struct { x: int; };= and =S2 = struct { x: int; };= are distinct types, so =const a: S1 = S2{...}= is a type error. Struct type names appear in error messages and LLVM IR instead of generic labels.

** Multi-name shorthand
Instead of defining each field on a different line like this
#+begin_src haste
const Vec3 = struct {
	x: float = 1;
	y: float = 1;
	z: float = 1;
};
#+end_src

You can use the short hand syntax instead
#+begin_src haste
const Vec3 = struct {
	x, y, z: float = 1;
};
#+end_src

** Auto-typed fields

A field type can be inferred from its default value using =auto=:

#+BEGIN_SRC haste
const Vec2 = struct {
    x: auto = 6.0;   // infers x: float
    y: auto = 7.0;   // infers y: float
};
#+END_SRC

The =:= shorthand is equivalent:

#+BEGIN_SRC haste
const Point = struct {
    x := 6.0;         // same as x: auto = 6.0
    y := 7.0;         // same as y: auto = 7.0
};
#+END_SRC

Inference follows the literal types — =0= becomes =int=, =0.0= becomes =float=, and ="hi"= becomes =string=. A field declared as =name: auto;= without a default value is rejected with "Cannot infer type for field".

that also works with the multi-name shorthand
#+begin_src haste
const Point = struct {
	x, y: auto = 6.7;
};

const Point2 = struct {
	x, y := 6.7;
};
#+end_src

* Struct literals
Struct values are created with brace syntax:

#+BEGIN_SRC haste
const pos = Vec2{ x: 0, y: 0 };          // literal with explicit type
const pos: Vec2 = .{ x: 1 };              // anonymous literal, type from annotation
const pos = .{ x: 1, y: 2 };             // anonymous, creates fresh struct type
#+END_SRC

=Type{ field: value, ... }= is a literal with an explicit struct type. =.{}= with a type annotation (=const x: T = .{...}=) injects the target type before field analysis, so fields are validated against the target. =.{}= without a type annotation creates a fresh anonymous struct type. Empty =.{}= creates an empty struct type. Omitting a field uses its declared default value.

Now lets suppose we have a struct =Foo= defined as below. as you can see, field =a= has a default value of =1=. but =b= is not.
#+begin_src haste
const Foo = struct {
	a: int = 1;
	b: float;
};
#+end_src

If you used the initializer below you are required to set the non-default fields. otherwise you get an error
#+begin_src haste
const foo = Foo{
	b: 6,
};

const foo2 = Foo {}; // you get an error
#+end_src

If you declared a variable. and you specified the type without specifying a value. haste will set the fields with default values to their default. but those who doesn't have a default value will be set to zero.
#+begin_src haste
const foo: Foo; // here `a` will have a value of 1
                // but `b` will have a value of 0
#+end_src

If you wanna explicitly set everything in the struct to zero. you can just assign =0= to it.
#+begin_src haste
const foo: Foo = 0;
// you can also use the `cast` operator
const foo2: Foo = cast 0;
const foo3 = cast[Foo]0;

// this also works? I didn't expect it to work at first actually. but it did!
const a = 0;
const foo4: Foo = cast a;
#+end_src

** Auto struct types
#+BEGIN_SRC haste
const Point = .{ x: 1, y: 2 };           // type: { x: int, y: int }
const empty = .{};                         // type: {}
const nested = .{ a: .{ b: "deep" } };   // nested anonymous structs
const copy = nested;                       // same type as nested
#+END_SRC

Each auto-struct literal gets a unique anonymous type. Auto-struct values can be copied, preserving type identity. When assigned to a typed variable, the target type is injected before field analysis. Auto-structs can also implicitly coerce to predefined struct types.

* Member access
Fields of a struct value are accessed with the dot (=.=) operator:

#+BEGIN_SRC haste
const Vec2 = struct {
    x: int;
    y: int;
};
const v = Vec2{ x: 10, y: 20 };
const a = v.x;    // 10
const b = v.y;    // 20
#+END_SRC

Member access works on any expression that produces a struct value — named structs, copied structs, and nested fields:

#+BEGIN_SRC haste
const Inner = struct { name: string; };
const Outer = struct {
    label: string;
    inner: Inner;
};
const o = Outer{
    label: "top",
    inner: .{ name: "nested" },
};

const inner = o.inner;       // Inner{ name: "nested" }
const name = o.inner.name;   // "nested"
#+END_SRC

Multiple levels of nesting are supported:

#+BEGIN_SRC haste
const A = struct { v: int; };
const B = struct { a: A; };
const C = struct { b: B; };
const c = C{ b: .{ a: .{ v: 7 } } };
const x = c.b.a.v;    // 7
#+END_SRC

Struct field access also works on auto-struct (=.{...}=) values:

#+BEGIN_SRC haste
const s = .{ a: 1, b: 2 };
const x = s.a;    // 1
const y = s.b;    // 2
#+END_SRC

Accessing a non-existent field is a compile-time error:

#+BEGIN_SRC haste
const v = Vec2{ x: 1, y: 2 };
const a = v.z;    // Error: Cannot access field 'z'. no such field inside 'Vec2'
#+END_SRC

Accessing a field on a non-struct type is also a compile-time error:

#+BEGIN_SRC haste
const x: int = 5;
const a = x.foo;  // Error: Cannot access field 'foo'. no such field inside 'int'
#+END_SRC

* Compile-time evaluation
Haste evaluates =const= declarations at compile time whenever possible. Arithmetic on literal values is constant-folded, forward references across top-level declarations are supported through hoisting, and type aliases resolve at compile time.

#+BEGIN_SRC haste
const a = 10 + 20;   // 30, computed at compile time
const b = a * 2;      // 60, depends on a
#+END_SRC

Forward references allow declarations to appear in any order:

#+BEGIN_SRC haste
const a = b + 1;     // 5.0
const b = c * 2;     // 4.0
const c: float = 2;
#+END_SRC

Recursive declarations such as =const S = struct { x: S; };= are detected and produce a compile-time error.

* Examples
#+BEGIN_SRC haste
const a: int = 42;
const b: float = 3.14;
const c: string = "hello";
const d: cstr = "world";

const x = 10 + 20;   // 30
const y = 50 - 15;   // 35
const z = 6 * 7;     // 42

const m: float = 10 + 3.5;   // 13.5
const n: float = 7.5 - 2;    // 5.5

const p: int;       // 0
const q: float;     // 0.0
const r: string;    // ""

const Number = int;
const s: Number = 42;
const t = cast[Number]3.14;   // 3
#+END_SRC
