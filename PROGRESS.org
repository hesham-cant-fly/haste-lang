#+TITLE: Haste Language — Implemented Features
#+DATE: 2026-05-13

* Declarations

** =const= — Immutable binding

#+BEGIN_SRC haste
const name = value;
const name: type = value;
const name: type;
#+END_SRC

- Without a value, the type must be specified; the default for that type is used
- =const= with a =type= value creates a type alias: =const MyInt = int=

** =var= — Mutable binding

#+BEGIN_SRC haste
var name = value;
var name: type = value;
var name: type;
#+END_SRC

- Same syntax as =const= (mutable, though only =const= is fully tested at global scope)

* Types

** Primitive types

| Type | Description | Default |
|------+-------------+---------|
| =int= | Signed 32-bit integer | =0= |
| =float= | 32-bit float | =0.0= |
| =string= | String slice =(data_ptr, length)= | empty string |
| =cstr= | Null-terminated C string pointer | empty string |
| =void= | No value | — |
| =type= | The type of types | — |

** Untyped literal types (no keyword — arise from literals)

| Type | From | Behavior |
|------+------+----------|
| =untyped_int= | =42= | Coerces to =int= or =float= on assignment |
| =untyped_float= | =3.14= | Coerces to =float= on assignment |
| =untyped_string= | ="hello"= | Coerces to =string= or =cstr= on assignment |

- =auto= in type position means "infer from the initializer"

* Type rules

** Arithmetic

- Typed + typed of different types → error (=int + float= is rejected)
- Untyped + untyped → untyped result (=untyped_int + untyped_float → untyped_float=)
- Typed + untyped → typed result (untyped coerces)
- Division by zero → compile-time error

** Assignment compatibility

| Declared type | Accepts |
|---------------+---------|
| =int= | =int=, =untyped_int== |
| =float= | =float=, =untyped_int=, =untyped_float= |
| =string= | =string=, =untyped_string= |
| =cstr= | =cstr=, =untyped_string= |
| =auto= | anything |

** Allowed casts (=cast[Target]source=)

| Source | Target |
|--------+--------|
| =untyped_int= | =int=, =float= |
| =untyped_float= | =float=, =int= |
| =int= | =float= |
| =float= | =int= |
| =untyped_string= | =string=, =cstr= |
| =string= | =cstr= |
| =cstr= | =string= |

* Compile-time evaluation

- All =const= declarations with literal or constant-foldable values are evaluated at compile time
- Arithmetic on literals is constant-folded
- Forward references across top-level declarations are supported (hoisting)
- Type aliases resolve at compile time
- Recursive declarations (e.g. =const S = struct { x: S; };=) are detected at compile time.

** Struct types

#+BEGIN_SRC haste
const Vec2 = struct {
    x: float;
    y: float = 0;          // default value
};
#+END_SRC

- Fields are declared as =name: type;= with an optional == default= value
- A struct type =is a type= value. =typeof(Vec2)= returns =type=
- Struct types are **nominally typed**: each struct declaration creates a unique type, even if two declarations have identical fields
  - =const S1 = struct { x: int; y: int; };= and =const S2 = struct { x: int; y: int; };= are /distinct types/ — =const a: S1 = S2{...}= is a type error
- Struct type names appear in error messages and LLVM IR instead of generic =struct= labels
  
** Struct literals

#+BEGIN_SRC haste
const pos = Vec2{ x: 0, y: 0 };          // named type literal
const pos: Vec2 = .{ x: 1 };              // anonymous literal, type from annotation
const pos: Vec2 = auto{ x: 1 };           // same as .{} with type annotation
const pos = .{ x: 1, y: 2 };             // anonymous literal, creates struct from fields
const pos = auto{ x: 1, y: 2 };          // same as .{} without type annotation
const pos = Vec3{};                        // all defaults
const pos: Vec3;                           // uninitialized → zero values
#+END_SRC

- =Type{ field: value, ... }= — literal with explicit type
- =.{}= / =auto{}= with type annotation (=const x: T = .{...}=) — target type is injected before analysis, fields validated against target type
- =.{}= / =auto{}= without type annotation — auto-struct literal; creates a fresh anonymous struct type with fields determined by the literal (see [[Auto struct types (=.{}= / =auto{}=)]])
- Auto-struct literals also work nested inside typed struct literal fields:
  - =Outer{ inner: .{ name: "x" } }= — =Inner= type is injected automatically onto the nested auto-struct
  - =Outer{ inner: auto{ name: "x" } }= — same behavior
- Empty =.{}= / =auto{}= creates an empty struct type
- Omitting a field uses its declared default (or the base type's default)
- =const x: StructType;= produces the zero value for all fields

** Auto struct types (=.{}= / =auto{}=)

#+BEGIN_SRC haste
const Point = .{ x: 1, y: 2 };            // type: { x: int, y: int } (auto-struct)
const Point = auto{ x: 1, y: 2 };         // same as .{}
const empty = .{};                         // type: {} (empty auto-struct)
const nested = .{ a: .{ b: "deep" } };   // type: { a: { b: string } }
const copy = nested;                       // same auto-struct type as nested

const Vec2 = struct {
	x: float;
	y: float;
};

const d = auto{
	x: 1,
	y: 2
};

const pos: Vec2 = d;

// the `cast` operator also works
const vel = cast[Vec2]d;
#+END_SRC

- Auto-struct literals (=.{}= or =auto{}=) assign each auto-struct literal a unique anonymous type (=HASTE_TY_AUTO_STRUCT=)
- Nested auto-structs produce nested anonymous types
- Auto-struct values can be copied (=const copy = original=), preserving type identity
- When an auto-struct is assigned to a typed variable or used as a typed struct field value, the target type is injected before field analysis:
  - =const a: Vec2 = .{ x: 1, y: 2 };= — the =Vec2= type is injected, fields validated against =Vec2=
  - =Outer{ inner: .{ name: "x" } }= — =Inner= type is injected onto the nested auto-struct
- Unannotated auto-struct literals create fresh types; they are /not/ compatible with any other struct type, even structurally identical ones
- auto-struct can implicitly coerce to predefined struct types.

* Examples

** Constants with explicit types

#+BEGIN_SRC haste
const a: int = 42;
const b: float = 3.14;
const c: string = "hello";
const d: cstr = "world";
#+END_SRC

** Compile-time arithmetic

#+BEGIN_SRC haste
const a = 10 + 20;   // 30
const b = 50 - 15;   // 35 (untyped)
const c = 6 * 7;     // 42
const d = 100 / 4;   // 25
#+END_SRC

** Mixed untyped/typed arithmetic

#+BEGIN_SRC haste
const a: float = 10 + 3.5;   // 13.5
const b: float = 7.5 - 2;    // 5.5
#+END_SRC

** Type casts

#+BEGIN_SRC haste
const a: float = 3.14;
const b: int = cast[int]a;           // 3
const c: float = cast[float]42;      // 42.0
const d: string = cast[string]"hello";
const e: cstr = cast[cstr]"world";
const f: string = cast[string]e;     // cstr → string
const g: cstr = cast[cstr]d;         // string → cstr
#+END_SRC

** Default values

#+BEGIN_SRC haste
const a: int;       // 0
const b: float;     // 0.0
const c: string;    // ""
const d: cstr;      // ""
#+END_SRC

** Type alias

#+BEGIN_SRC haste
const Number = int;
const a: Number = 42;
const b = cast[Number]3.14;   // 3
#+END_SRC

** Forward references

#+BEGIN_SRC haste
const a = b + 1;     // 5.0
const b = c * 2;     // 4.0
const c: float = 2;
#+END_SRC
