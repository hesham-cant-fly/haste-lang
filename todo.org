*** DONE Fix token-boundary bug in keyword matching
    =source/scanner.c:160-165= uses =matches()= which calls =strncmp=

    overlong literals.

    =source/codegen.c:67= accesses =node->variable.value->value= without

*** DONE Fix =--help= exit code
    =source/options.c:35= returns =ERROR= (exit 1) after printing help.
    Should return =OK= (exit 0).

** Undefined Behavior / Memory Safety
*** DONE =get_line_number= walks backwards past buffer
    =source/error.c:30-31= iterates =current -== 1= with no lower bound
    check. Assumes a ='\0'= exists before the string, which will walk
    past the buffer start into undefined memory.

*** DONE =span_to_trimed= accesses =start[-1]= when =len= is 0
    =source/span.c:12= checks =result.len > 0= AFTER indexing into
    =result.start[result.len - 1]=. Short-circuit eval means this is UB
    when =len= is 0. Swap the conditions.

*** DONE =get_pre_eof= underflows on short token list
    =source/parse.c:48= accesses =tokens.items[tokens.len - 2]=. If the
    token list has 0 or 1 tokens (empty file), this underflows the
    array.

*** DONE =print_token= indexes without bounds check
    =source/token.c:43= uses =TOKEN_KIND_LIT[token.kind]= directly.
    =TK_IDENT= starts at 1, so an uninitialized (=0=) token kind reads
    garbage.

*** DONE =in_range= uses =memcpy= of =-1= as sentinel
    =source/unicode.c:101-104= builds a uint32 sentinel by memcpy-ing
    =int32_t(-1)=. Implementation-defined behavior. Use =UINT32_MAX= or
    =0xFFFFFFFF= directly.

** Logic Bugs
*** DONE =_check= uses =va_arg= with enum
    =source/parse.c:77= uses =va_arg(args, enum token_kind)= — enums are
    promoted to =int= in variadic calls. The sentinel =(enum
    token_kind){0}= may not compare correctly on all platforms.

*** DONE =get_current_working_directory= ignores allocator on reuse
    =source/source.c:15-16= caches CWD in a =static=, returning the old
    pointer on subsequent calls regardless of which allocator is passed.

** Error Handling
*** DONE =read_entire_file= exits silently on failure
    =source/source.c:65= calls =Exit(1)= with no error message when
    =fopen= fails (e.g. file not found, permission denied). User gets
    zero feedback. Same at =source/source.c:75= for =fread= failures.

*** DONE =read_entire_file= uses text mode
    =source/source.c:64= opens with ="r"= (text mode) instead of ="rb"=.

    counts.

** Code Quality
*** DONE Block comment scan is O(n²) worst case
    =source/scanner.c:134-138= uses =strstr= on the entire rest of the
    file to find ="*/"=. A simple character-by-character scan is simpler
    and equally fast.

*** DONE No cycle detection in declaration analysis
    =source/analysis.c:79-81= sets =decl->analyzing = true= but never
    checks it before recursing. Mutually recursive =find_local_first=
    calls will cause infinite recursion instead of a clean error.

*** DONE Inconsistent token ownership on error paths
    =source/main.c:85-89= (scan error) relies on the scanner having
    freed its own tokens internally, but =source/main.c:100-104= (parse
    error) explicitly frees tokens. Inconsistent: sometimes callee
    frees, sometimes caller.

*** DONE Un-namespaced macros in =my_array.h=
    =source/haste.h= includes =include/my_array.h= which defines
    =INITIAL_CAP= and =GROW_FACTOR= without namespacing. Any other code
    defining these will silently break the dynamic array macros.
