const main() {
  @println("Hello, World!"); // anything that starts with `@` is consider as macro expantion

  const mini_program = "++-++++** / * *";
  var result: fsize = 0;

  for ch in mini_program {
    match ch {
      case '+' do result += 1;
      case '-' do result -= 1;
      case '*' do result *= 2;
      case '/' do result /= 2;
      else {} // Ignore anything else
    }
  }

  @println("Result: {}", result);
}

* Inspirations
Haste is heavily inspired by C, Zig, and Odin, along with a few other languages. C is my all-time favorite, with Zig being a close second. Haste‚Äôs simplicity and low-level nature come from my background in C. I‚Äôve never been a fan of C++ or its complexity, so Haste is designed to be closer in spirit to C and Zig. think of it as C with methods and a sane type system.

The idea of treating types as values is borrowed from Zig. I find this model ideal for reflection and it significantly simplifies the compiler‚Äôs implementation. From Odin, I drew inspiration from its type system, the =context= model and overall design philosophy, which encouraged me to focus on building the language I want, rather than trying to please everyone.
